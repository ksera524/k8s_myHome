# DR/バックアップ

## 目的
- 重要データと設定の復旧性を確保し、停止時間を最小化する

## 方針
- etcd と重要 namespace のバックアップを定期化
- 復旧手順を自動化し、定期的に検証

## 対応内容
- バックアップ対象/保存先の定義
- スナップショット取得と保管の自動化
- リストア手順の文書化と検証フロー

## 変更対象（候補）
- `automation/scripts/`
- `manifests/platform/`（CronJob or Velero）
- `docs/operations-guide.md`

## 受け入れ基準
- 定期バックアップが実行される
- 復旧手順が手順書化されている
- 検証ログが残る
